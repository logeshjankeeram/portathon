Detection Methods

Settings live in methods.json. Combine multiple methods for robust detection; thresholds should be tuned to local vessel profiles and sensor layouts.

MultipleReceiversMethod (Pairwise Distance Monitoring – PDM)

Why: Genuine receivers mounted on different platforms should not collapse toward the same spoofed position. How: Monitor pairwise distances and flag when observed distance is lower than a ratio of the ship-to-ship baseline.

{
  "distances": {
    "192.168.0.10, 192.168.0.11": 4
  },
  "distance_ratio_thresholds": {
    "192.168.0.10, 192.168.0.11": 0.5
  }
}

Tuning: Use known mooring/operational distances as baselines; relax ratios when tugboats operate very close to vessels.

PhysicalSpeedLimitMethod (PCC_sog)

Why: Spoofed trajectories often imply unrealistic speeds. How: Bound SOG by the vessel class capability. 

{
  "max_speed": 30
}

Tuning: Configure per ship type (tug, cargo, tanker) or per MMSI when available.

PhysicalRateOfTurnLimitMethod (PCC_rot)

Why: Sharp turns at low speed can be sensor noise; at higher speeds unrealistic ROT suggests spoofing. How: Apply ROT bounds only above a minimum speed.

{
  "max_rate_of_turn": 7.5,
  "min_speed_to_determine_rate_of_turn": 15
}

Tuning: Increase min_speed to reduce false positives in harbor maneuvers.

PhysicalHeightLimitMethod (PCC_height)

Why: Maritime platforms near sea level should not fluctuate significantly in altitude. How: Clamp allowed height range near 0 m AMSL.

{
  "min_height": -1,
  "max_height": 1
}

Tuning: Expand to ±2–3 m if barometric/vertical errors are larger in your sensors.

PhysicalEnvironmentLimitMethod (PCC_env)

Why: Valid maritime positions should lie on water masks. How: Reject points on land using a raster mask.

Tuning: Use higher-resolution water_map.png near complex coastline; precompute tiles for performance.

OrbitPositionsMethod (EDV)

Why: Inconsistent satellite geometry is a hallmark of spoofing. How: Compare observed azimuth/elevation to predicted ephemerides; ensure minimum elevation and small deviations. 

{
  "min_elevation": 5,
  "allowed_azimuth_deviation": 1,
  "allowed_elevation_deviation": 1
}

Maintenance: Keep gps.tle updated; deviations may need loosening under multipath.

TimeDriftMethod (CDM)

Why: Spoofing can perturb time solutions. How: Regress device clock drift; alert when slope exceeds tolerance.

{
  "max_clock_drift_dev": 0.1
}

Tuning: Increase tolerance for older receivers with noisier oscillators.

CarrierToNoiseDensityMethod (CNM)

Why: Fake signals often present abnormal C/N₀ distributions. How: Enforce min/max C/N₀ bounds per satellite/channel. 

{
  "min_carrier_to_noise_density": 20,
  "max_carrier_to_noise_density": 50
}

Tuning: Tailor by local RF environment; consider different bounds for sheltered basins vs open sea.

Fusion & Thresholding

The DetectionHandler aggregates method outputs into a spoofing indicator. The system uses a configurable detection_threshold to trigger events and feeds both detections and benign state updates to the UI for continuous feedback.
